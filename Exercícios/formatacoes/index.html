As formatações em HTML5

Introdução

O HTML5 trouxe uma mudança de lógica em relação ao HTML4. Antes dele, as tags usadas para formatar um texto eram muitas vezes baseadas em formato, ou seja, em como ia ficar esse conteúdo na página. Para colocar um itálico, por exemplo, a tag usada era <i>. Com o desenvolvimento das CSS, a formatação das páginas ficou muito mais poderosa. Isso talvez tenha liberado as tags em HTML para serem mais lógicas, um modelo chamado de semântico. Isso significa que as tags passam a representar mudanças de conteúdo, ou seja, de significado, de sentido. No HTML5, o que chamamos de itálico não é entendido apenas do ponto de vista da forma, mas sim do seu significado: ele é um destaque. Assim, como veremos a seguir, a tag atualmente usada é <em>, que vem do inglês emphasis, ou seja, ênfase.

Principais formatações

Negrito / Destaque

A forma semântica de dar destaque é usando a tag <strong>.

Itálico / Ênfase

A forma semântica de dar ênfase é usando a tag <em>.

Texto marcado

Para um texto tipo marca-texto, usamos a tag <mark>.

Texto pequeno

Para usar as famosas letras miúdas, usamos a tag <small>. No HTML5, não existe tag para um texto grande. Isso se faz com CSS.

Texto deletado

Para riscar um texto, dando o significado que é para ele ser desconsiderado do sentido final, usamos a tag <del>.

Texto inserido

Já o sublinhado é considerado um texto inserido, como se houvesse um espaço em que foi acrescentado um texto depois. A tag a ser usada é <ins>.

Texto sobrescrito

É utilizado para as colocar caracteres ou palavras menores e mais acima das restantes, como no caso de notações matemáticas: 102. Usamos a tag <sup> pra isso.

Texto subscrito

É utilizado para colocar caracteres ou palavras menores e mais abaixo que as demais, como no caso de notações químicas: H20. Para isso, usamos a tag <sub>.

Outras formatações

Código-fonte e Pré-formatação

Para dar mais legibilidade a um texto de código, como por exemplo (de um código em C) printf("ciphertext: ");, usamos a tag <code>.

No entanto, quando queremos citar códigos que tenham identação, a tag <code> não vai ser suficiente, já que ela ignora qualquer identação e coloca tudo na mesma linha. Uma forma de lidar com isso é envelopar tudo o que estiver já envelopado por <code> em uma tag <pre>. Ela vai fazer com que todo espaço ou troca de linha no código seja levado para o navegador. Citações

Para citar algum texto ou alguém, usamos a tag <q>, que vem de quote, ou seja, citação. Por exemplo: É difícil lidar com gente burra. (Clarice Lispector ou Mário Quintana).

Tudo o que essa tag faz, a princípio, é colocar entre áspas. Por que não simplesmente digitar as áspas, então? Acredito que tenha a ver com a declaração dessa parte como citação. Assim, se vamos mudar o estilo das citações em uma página, podemos usar CSS para alterar todas as inserções da tag <q>.

Já para fazer uma citação em bloco, usamos a tag <blockquote>. Nela, podemos usar um parâmetro cite para indicar o link da fonte - o que não vai aparecer no navegador, mas vai auxiliar os mecanismos de busca e a parte interna do site a entender da onde vem a citação.
